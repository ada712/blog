    高度坍塌

    ada 2021/12/12

    # 背景：

    • 父元素的高度，都是由内部 未浮动 子元素的高度撑起的
    • 子元素浮动，还会对父元素造成影响

    如果子元素浮动起来，就不占用普通文档流的位置。父元素高度就会失去支撑，也称为高度坍塌。

    # 问题现象：

    父元素不写高度时，子元素写了浮动后，父元素会发生高度塌陷（造成父元素高度为 0）

    # 解决方案

    # 方案一

    • 为父元素设置overflow:hidden属性
    • 原理: CSS中的overflow:hidden属性会强制要求父元素必须裹住所有内部浮动的元素，以及所有元素的margin范围
    • 缺点: 不能和position定位配合使用，超出的尺寸会被隐藏

    # 方案二

    • 在浮动元素下方添加空div，并给元素声明 clear：both，保险起见，再加height:0。清除个别块元素可能自带的height:16px;
    • 原理: 利用clear:both属性和父元素必须包含非浮动的元素两个原理
    • 缺点：需要添加多余的空标签并添加属性

    # 方案三

    • 设置父元素也浮动
    • 原理：浮动属性也会强制父元素扩大到包含所有浮动的内部元素
    • 缺点： 产生了新的浮动，无穷无尽

    # 方案四

    • 父元素::after{ content:""; display:table; clear:both; height:0 }
    • ps : 个别浏览器，display:table，可能带默认高度保险起见: 再加一个属性height:0px
